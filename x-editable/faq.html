常见的问题

怎么与服务器交互?

X-editable 对服务端端语言没有限制，你可以写任何的服务端语言。
后端常用逻辑:

  1. 读取请求传递的字段 pk, name 和 value (默认POST)。
  2. 验证value:
  3. 如果 验证失败 - 返回 HTTP status != 200 (e.g. 400 Bad Request) 和返回错误信息。
  4. 如果 验证成功 - 按照业务逻辑处理和返回 HTTP status = 200 。

服务端未获取到提交数据?

检查传递字段 pk 和 url 是否为空。
如果所有修改内容主键是唯一可以全局设置pk字段固定传递。

如果没有更改值，表单不会提交，可设置 savenochange 选项

如何更新提交给服务端的内容?

定义 params方法:

$('#field').editable({
    ...        
    params: function(params) {  //重新给 `name`, `value` , `pk` 字段赋值
        var data = {};
        data['id'] = params.pk;
        data[params.name] = params.value;
        return data;
    }
});

如何给提交的json转换成字符串?

定义 params 方法:

//submit in JSON format        
$('#field').editable({
    ...        
    params: function(params) {
        return JSON.stringify(params);  //note: for older browsers you may need json2.js from json.org
    }
});

如何自定义X-editable样式?

重写css样式, e.g.

a.editable-click {
    color: green;
    border-color: green;
} 
a.editable-click.editable-disabled {
    color: black;  
    border-bottom: none;
}
a.editable-empty
    color: gray;
    font-style: italic;
}
